🔍 Tổng quan & Cách vận hành

git2hdd giải quyết bài toán: "Làm sao backup dự án từ SSD sang nhiều HDD một cách tự động, đáng tin cậy?"

Thay vì duy trì nhiều file .bat rời rạc (mỗi HDD một file), git2hdd gộp tất cả vào một CLI tool duy nhất với 2 chế độ backup:

Chế độ 1: Git Backup Hàng ngày

Sử dụng Git để backup toàn bộ lịch sử commit. Quy trình:

1
git add .
Stage tất cả thay đổi
→
2
git commit
Tạo commit mới
→
3
git push
Push đến tất cả HDD
💡 Ưu điểm Git Backup Lưu được toàn bộ lịch sử commit, dễ dàng rollback, mỗi HDD là một bare repo độc lập — nếu 1 HDD hỏng, các HDD khác vẫn còn dữ liệu.

⚙️ Yêu cầu hệ thống

Yêu cầu Phiên bản Ghi chú
Hệ điều hành Windows 10+ Cảnh báo nếu chạy trên OS khác
Node.js ≥ 16.0.0 Kiểm tra tự động khi chạy
Git for Windows Bất kỳ Tải tại đây

📦 Cài đặt

Cách 1: Cài đặt toàn cục (khuyến nghị)

$ npm install -g git2hdd

Sau đó sử dụng trực tiếp từ bất kỳ đâu:

$ git2hdd --version
1.0.0

Cách 2: Chạy trực tiếp từ source

$ git clone https://github.com/git2hdd/git2hdd.git
$ cd git2hdd
$ npm install
$ node bin/git2hdd.js --version

Kiểm tra cài đặt

$ git2hdd --help
Tự động hóa Git backup từ SSD sang nhiều HDD trên Windows

Options:
  -v, --version   Hiển thị phiên bản hiện tại
  -h, --help      Hiển thị trợ giúp

Commands:

  setup-remotes   Tạo bare Git repositories trên các HDD đích
  add-remotes     Thêm các HDD làm Git remote vào Source_Repo
  backup          git add . → git commit → git push đến tất cả HDD
  config          Xem và chỉnh sửa cấu hình git2hdd.config.json
  log             Xem lịch sử các lần backup/mirror

🚀 Lệnh init Chạy 1 lần

Khởi tạo Git repository tại thư mục nguồn và tạo file cấu hình git2hdd.config.json.

Cú pháp

$ git2hdd init --source <đường_dẫn>

Tham số

Tham số Bắt buộc Mô tả
--source Không Đường dẫn thư mục nguồn. Mặc định: thư mục hiện tại

Chi tiết hoạt động

  1. Kiểm tra Git đã được cài đặt chưa
  2. Kiểm tra đường dẫn source có tồn tại không
  3. Chạy git init tại thư mục source (nếu chưa có .git)
  4. Tạo file git2hdd.config.json với cấu hình mặc định
  5. Hiển thị gợi ý chạy setup-remotes tiếp theo

Ví dụ

# Chỉ định thư mục nguồn
$ git2hdd init --source D:\work\my-project

✓ Đã khởi tạo Git repository tại D:\work\my-project
✓ Đã tạo file git2hdd.config.json

Bước tiếp theo: git2hdd setup-remotes --targets E:\backup\my-project.git

# Hoặc dùng thư mục hiện tại
$ cd D:\work\my-project
$ git2hdd init
⚠️ Lưu ý Nếu thư mục đã có .git, lệnh sẽ bỏ qua bước git init và chỉ tạo file config. Nếu file config đã tồn tại, bạn sẽ được hỏi có muốn ghi đè không.

🗄️ Lệnh setup-remotes Chạy 1 lần

Tạo các bare Git repositories trên các HDD đích. Bare repo là repo không có working directory — chỉ chứa dữ liệu Git, phù hợp làm remote backup.

Cú pháp

$ git2hdd setup-remotes --targets   ...

Tham số

Tham số Bắt buộc Mô tả
--targets Có Danh sách đường dẫn HDD đích (1–10 đường dẫn). Nên kết thúc bằng .git

Chi tiết hoạt động

  1. Đọc file git2hdd.config.json để lấy cấu hình
  2. Với mỗi target:
    • Tạo thư mục nếu chưa tồn tại
    • Chạy git init --bare
  3. Cập nhật danh sách targets trong config

Ví dụ

# Tạo bare repo trên 3 HDD
$ git2hdd setup-remotes --targets \
    E:\backup\my-project.git \
    F:\backup\my-project.git \
    G:\backup\my-project.git

✓ Đã tạo bare repo tại E:\backup\my-project.git
✓ Đã tạo bare repo tại F:\backup\my-project.git
✓ Đã tạo bare repo tại G:\backup\my-project.git
✓ Đã cập nhật git2hdd.config.json
💡 Mẹo Nên đặt tên thư mục kết thúc bằng .git (ví dụ: E:\backup\my-project.git) theo quy ước của Git bare repository.

🔗 Lệnh add-remotes Chạy 1 lần

Thêm các HDD đã setup làm Git remote vào source repository. Mỗi HDD sẽ được đặt tên tự động: hdd1, hdd2, hdd3,...

Cú pháp

$ git2hdd add-remotes

Chi tiết hoạt động

  1. Đọc danh sách targets từ config
  2. Đọc tiền tố remote từ config (remotePrefix, mặc định: hdd)
  3. Với mỗi target:
    • Nếu remote chưa tồn tại → git remote add hdd1
    • Nếu remote đã tồn tại → git remote set-url hdd1

Ví dụ

$ git2hdd add-remotes

✓ Đã thêm remote hdd1 → E:\backup\my-project.git
✓ Đã thêm remote hdd2 → F:\backup\my-project.git
✓ Đã thêm remote hdd3 → G:\backup\my-project.git

# Kiểm tra kết quả
$ git remote -v
hdd1    E:\backup\my-project.git (fetch)
hdd1    E:\backup\my-project.git (push)
hdd2    F:\backup\my-project.git (fetch)
hdd2    F:\backup\my-project.git (push)
hdd3    G:\backup\my-project.git (fetch)
hdd3    G:\backup\my-project.git (push)
ℹ️ An toàn khi chạy lại Nếu remote đã tồn tại, lệnh sẽ cập nhật URL thay vì báo lỗi. Bạn có thể chạy lại lệnh này bất kỳ lúc nào.

💾 Lệnh backup Hàng ngày

Lệnh UN] Sẽ thực hiện: 1. git add . 2. git commit -m "Backup 2026-06-02 14:30:00" 3. git push hdd1 main 4. git push hdd2 main 5. git push hdd3 main

⚠️ Xử lý partial failure Nếu một HDD push thất bại (ví dụ: HDD không kết nối), các HDD khác vẫn tiếp tục backup. Lệnh sẽ thoát với exit code 8 và hiển thị HDD nào thất bại.

⚙️ Lệnh config Quản lý

Xem, cập nhật, hoặc reset file cấu hình git2hdd.config.json.

Cú pháp

$ git2hdd config --show
$ git2hdd config --set  --value 
$ git2hdd config --reset

Tham số

Tham số Mô tả
--show Hiển thị cấu hình hiện tại (mặc định nếu không có flag nào)
--set Tên trường cần cập nhật (hỗ trợ dot-notation)
--value Giá trị mới (dùng với --set)
--reset Reset toàn bộ cấu hình về mặc định

Ví dụ

# Xem cấu hình hiện tại
$ git2hdd config --show

{
  "sourcePath": "D:\\work\\my-project",
  "targets": [
    "E:\\backup\\my-project.git",
    "F:\\backup\\my-project.git"
  ],
  "defaultBranch": "main",
  "remotePrefix": "hdd"
}

# Đổi branch mặc định
$ git2hdd config --set defaultBranch --value master

# Reset về mặc định
$ git2hdd config --reset

📋 Lệnh log Quản lý

Xem lịch sử các lần backup và mirror. Dữ liệu được lưu ở định dạng JSON Lines (git2hdd.log).

Cú pháp

$ git2hdd log
$ git2hdd log --lines 50

Tham số

Tham số Mặc định Mô tả
-n, --lines 20 Số bản ghi gần nhất cần hiển thị

Ví dụ

$ git2hdd log

📋 Lịch sử backup (10 bản ghi gần nhất)

┌─────────────────────┬─────────┬──────────────────────────────┐
│ Thời gian           │ Loại    │ Kết quả                      │
├─────────────────────┼─────────┼──────────────────────────────┤
│ 2026-06-02 14:30:00 │ backup  │ 3/3 HDD ✅                   │

📄 File cấu hình git2hdd.config.json

File này được tạo tự động bởi lệnh init và nằm tại thư mục gốc của dự án.

Cấu trúc đầy đủ

{
  "sourcePath": "D:\\work\\my-project",
  "targets": [
    "E:\\backup\\my-project.git",
    "F:\\backup\\my-project.git",
    "G:\\backup\\my-project.git"
  ],
  "defaultBranch": "main",
  "remotePrefix": "hdd"
}

Giải thích từng trường

Trường Kiểu Bắt buộc Mô tả
sourcePath string ✅ Đường dẫn tuyệt đối đến thư mục dự án trên SSD
targets string[] ✅ Danh sách đường dẫn bare repo trên các HDD (cho Git backup)
defaultBranch string Không Branch mặc định để push. Mặc định: "main"
remotePrefix string Không Tiền tố đặt tên remote. Mặc định: "hdd" → hdd1, hdd2,...
💡 Config được lưu ở đâu? File git2hdd.config.json nằm trong thư mục hiện tại khi bạn chạy lệnh init. Hãy đảm bảo chạy các lệnh tiếp theo từ cùng thư mục hoặc chỉ định đúng --source.

🚨 Exit codes

git2hdd sử dụng exit codes chi tiết để giúp bạn xác định loại lỗi khi chạy tự động (ví dụ: trong cron job hoặc CI):

Code Ý nghĩa Khi nào xảy ra
0 Thành công hoàn toàn Tất cả thao tác đều OK
1 Lỗi chung Lỗi không xác định
2 Lỗi cấu hình Config không tìm thấy hoặc không hợp lệ
3 Thiếu phụ thuộc Git không tìm thấy
4 Đường dẫn không tồn tại Source path hoặc target path không hợp lệ
5 Lỗi Git git init/commit/push thất bại
7 Node.js version quá cũ Node.js < 16.0.0
8 Partial failure Ít nhất 1 HDD thất bại, nhưng các HDD khác OK
ℹ️ Sử dụng exit code trong script 
REM Kiểm tra kết quả trong batch script
git2hdd backup
IF %ERRORLEVEL% EQU 0 ECHO Backup thành công!
IF %ERRORLEVEL% EQU 8 ECHO Có HDD bị lỗi, kiểm tra lại!

🔧 Xử lý sự cố

Lỗi thường gặp

❌ "git: command not found" hoặc exit code 3

Git chưa được cài hoặc không nằm trong PATH.

# Kiểm tra Git
$ git --version

# Nếu chưa có, tải tại: https://git-scm.com/download/win
# Khi cài, đảm bảo chọn "Add Git to PATH"

❌ "Config not found" hoặc exit code 2

File git2hdd.config.json không tồn tại.

# Chạy init trước
$ git2hdd init --source D:\work\my-project

❌ "Path does not exist" hoặc exit code 4

Đường dẫn source hoặc target không tồn tại.

# Kiểm tra đường dẫn trong config
$ git2hdd config --show

# Đảm bảo tất cả HDD đã được mount
$ dir E:\backup\

❌ Push thất bại (exit code 8)

Một hoặc nhiều HDD không thể push. Kiểm tra:

  • HDD có được kết nối không?
  • Bare repo có tồn tại không? (chạy lại setup-remotes)
  • Remote có đúng URL không? (chạy lại add-remotes)
# Kiểm tra remote URLs
$ git remote -v

# Thử push thủ công để xem lỗi chi tiết
$ git push hdd1 main --verbose

Debug mode

Để xem chi tiết hơn, bạn có thể kiểm tra log:

# Xem log gần nhất
$ git2hdd log --lines 5

# Hoặc đọc file log trực tiếp
$ type git2hdd.log

    F:\backup\MyGame.git \
    G:\backup\MyGame.git \

# Backup cuối ngày
$ git2hdd backup -m "Day 42: added boss fight"

Ví dụ 3: Sử dụng trong Task Scheduler (Windows)

@echo off
REM daily-backup.bat — Chạy hàng ngày qua Task Scheduler

REM Git backup
git2hdd backup
IF %ERRORLEVEL% NEQ 0 (
    ECHO Backup failed with code %ERRORLEVEL%
    REM Gửi thông báo hoặc ghi log lỗi
)
