💡 Hướng dẫn vận hành nhanh (Quick Guide)

1
Khởi tạo

Đảm bảo đã chạy lệnh git2hdd init tại thư mục dự án.

2
Cấu hình

Thêm các thư mục HDD backup và HDD mirror tại tab Cấu hình dự án.

3
Đăng ký Remote

Sau khi thêm HDD Backup mới, chạy lệnh git2hdd add-remotes tại terminal một lần.

4
Vận hành

Chọn tab Chạy Backup / Mirror để thực thi và giám sát tiến trình trực quan.

💾 Thực thi Git Backup (Hàng ngày)

Thực hiện: git add . ➜ git commit ➜ git push đến tất cả các ổ cứng HDD đã được cài đặt.

➕ Thêm dự án mới

Đăng ký một thư mục dự án (source repo trên SSD) để quản lý backup riêng cho từng dự án.

Đường dẫn tuyệt đối tới thư mục làm việc chính trên SSD của dự án.

📂 Danh sách dự án đang quản lý

Chọn một dự án để thao tác, hoặc gỡ khỏi danh sách quản lý (không xóa dữ liệu thật trên đĩa).

Chỉnh sửa cấu hình git2hdd.config.json

Đường dẫn tuyệt đối đến thư mục làm việc chính trên SSD của bạn.

Đường dẫn đến các bare repository trên HDD dùng để push dự phòng (thường kết thúc bằng .git).

Nhánh Git chính dùng để đồng bộ (thường là main hoặc master).

Tên các remote Git được đặt tự động. Mặc định là hdd (tạo thành hdd1, hdd2...).

📅 Lên lịch backup tự động

Sử dụng Windows Task Scheduler để chạy lệnh backup tự động hàng ngày.
